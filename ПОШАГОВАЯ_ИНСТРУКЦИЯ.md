## 🐙 Шаг 2: Создай GitHub репозиторий

### 2.1. Перейди на GitHub
Открой https://github.com/new

### 2.2. Заполни форму
- Repository name: `huddle-server`
- Description: `Huddle Messenger Server`
- Public (выбери Public)
- НЕ добавляй README, .gitignore, license

### 2.3. Нажми "Create repository"

### 2.4. Скопируй URL
Скопируй URL типа: `https://github.com/твой-username/huddle-server.git`

---

## 💻 Шаг 3: Загрузи код на GitHub

### 3.1. Открой терминал
В VS Code нажми `Ctrl + ~` или открой PowerShell

### 3.2. Перейди в папку сервера
```bash
cd Huddle/HuddleServer
```

### 3.3. Инициализируй Git
```bash
git init
```

### 3.4. Добавь все файлы
```bash
git add .
```

### 3.5. Сделай коммит
```bash
git commit -m "Initial commit: Huddle Server"
```

### 3.6. Добавь remote
```bash
git branch -M main
git remote add origin https://github.com/ТВОЙ-USERNAME/huddle-server.git
```

⚠️ **ВАЖНО:** Замени `ТВОЙ-USERNAME` на свой GitHub username!

### 3.7. Загрузи на GitHub
```bash
git push -u origin main
```

Если попросит логин/пароль:
- Username: твой GitHub username
- Password: используй Personal Access Token (не пароль!)
  - Создай токен: https://github.com/settings/tokens
  - Выбери "Generate new token (classic)"
  - Выбери срок действия и права (repo)
  - Скопируй токен и используй вместо пароля

---

## 🚂 Шаг 4: Деплой на Railway

### 4.1. Открой Railway
Перейди на https://railway.app

### 4.2. Войди через GitHub
Нажми "Login" → "Login with GitHub"

### 4.3. Разреши доступ
Railway попросит доступ к твоим репозиториям - разреши

### 4.4. Создай новый проект
Нажми "New Project"

### 4.5. Выбери репозиторий
- Нажми "Deploy from GitHub repo"
- Найди и выбери `huddle-server`
- Нажми "Deploy Now"

### 4.6. Жди деплоя
Railway автоматически:
- Обнаружит Dockerfile
- Соберет образ
- Запустит сервер
- Это займет 2-3 минуты

Ты увидишь логи сборки. Жди пока статус станет "Success" ✅

---

## 🌐 Шаг 5: Получи URL сервера

### 5.1. Открой настройки
В Railway нажми на свой проект → вкладка "Settings"

### 5.2. Сгенерируй домен
- Найди раздел "Networking"
- Нажми "Generate Domain"
- Railway создаст URL типа: `huddle-server-production.up.railway.app`

### 5.3. Скопируй URL
Скопируй этот URL - он понадобится!

---

## 🔧 Шаг 6: Обнови клиент

### 6.1. Открой ChatService.cs
Файл: `Huddle/Services/ChatService.cs`

### 6.2. Найди строку
```csharp
private readonly string _serverUrl = "http://localhost:5000/chathub";
```

### 6.3. Замени на свой URL
```csharp
private readonly string _serverUrl = "https://твой-url.railway.app/chathub";
```

⚠️ **ВАЖНО:** 
- Используй `https://` (не http://)
- Добавь `/chathub` в конце
- Замени `твой-url.railway.app` на свой URL из шага 5.3

### 6.4. Сохрани файл
Нажми `Ctrl + S`

## 🎉 Шаг 8: Проверка

### 8.1. Проверь сервер в браузере
Открой в браузере: `https://твой-url.railway.app/chathub`

Должна быть ошибка 404 или текст про SignalR - это НОРМАЛЬНО! ✅

### 8.2. Запусти клиент
```bash
./bin/Debug/net10.0-windows/Huddle.exe
```

### 8.3. Войди в аккаунт
Зарегистрируйся или войди

### 8.4. Отправь сообщение
- Найди пользователя через поиск
- Открой чат
- Отправь сообщение

### 8.5. Проверь логи Railway
В Railway перейди в "Deployments" → "View Logs"

Ты должен увидеть:
```
🚀 Huddle Server запущен на http://0.0.0.0:XXXX
📡 SignalR Hub: http://0.0.0.0:XXXX/chathub
```

## 🐛 Проблемы?

### Сервер не запускается
- Проверь логи в Railway (вкладка "Deployments")
- Убедись что все файлы загружены на GitHub

### Клиент не подключается
- Проверь URL в ChatService.cs
- Убедись что используешь `https://` (не http://)
- Проверь что добавил `/chathub` в конце

### "Connection refused"
- Подожди 1-2 минуты после деплоя
- Проверь что сервер запущен в Railway

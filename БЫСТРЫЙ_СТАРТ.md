# 🚀 Быстрый деплой на Railway (5 минут)

## Шаг 1: Создай GitHub репозиторий

1. Перейди на https://github.com/new
2. Назови репозиторий: `huddle-server`
3. Сделай его Public
4. Нажми "Create repository"

## Шаг 2: Загрузи код на GitHub

Открой терминал в папке `Huddle/HuddleServer` и выполни:

```bash
git init
git add .
git commit -m "Huddle Server"
git branch -M main
git remote add origin https://github.com/ТВОЙ-USERNAME/huddle-server.git
git push -u origin main
```

Замени `ТВОЙ-USERNAME` на свой GitHub username!

## Шаг 3: Деплой на Railway

1. Перейди на https://railway.app
2. Нажми "Login" → войди через GitHub
3. Нажми "New Project"
4. Выбери "Deploy from GitHub repo"
5. Выбери репозиторий `huddle-server`
6. Railway автоматически задеплоит сервер!

## Шаг 4: Получи URL сервера

1. В Railway нажми на свой проект
2. Перейди в "Settings" → "Networking"
3. Нажми "Generate Domain"
4. Скопируй URL (например: `huddle-server-production.up.railway.app`)

## Шаг 5: Обнови клиент

Открой файл `Huddle/Services/ChatService.cs` и измени строку:

```csharp
private readonly string _serverUrl = "http://localhost:5000/chathub";
```

На:

```csharp
private readonly string _serverUrl = "https://твой-url.railway.app/chathub";
```

Замени `твой-url.railway.app` на URL из шага 4!

## Шаг 6: Пересобери клиент

```bash
cd Huddle
dotnet build Huddle.csproj
```

## 🎉 Готово!

Теперь твой сервер работает 24/7 в интернете!

Запусти клиент и проверь - сообщения должны работать через интернет-сервер.

## ❓ Проблемы?

1. **Сервер не запускается** - проверь логи в Railway (вкладка "Deployments")
2. **Клиент не подключается** - убедись что URL правильный и начинается с `https://`
3. **Нужна помощь** - напиши мне!
